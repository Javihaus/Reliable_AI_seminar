\begin{abstract}
This recitation examines how response latency creates fundamental interaction constraints. We quantify bandwidth degradation, examine effect sizes across cognitive domains, and provide design guidance for latency-sensitive applications.
\end{abstract}

\section{The Latency Problem}

\begin{warningbox}
Response latency degrades interaction bandwidth by 56\% at 10-second delays. This is independent of response quality---even perfect answers suffer from timing issues.
\end{warningbox}

\section{Human Cognitive Architecture}

Humans have timing expectations:
\begin{itemize}
    \item Conversational: 200-500ms
    \item Simple queries: 1-2s
    \item Complex queries: 3-5s acceptable
    \item Beyond 10s: Significant disruption
\end{itemize}

\begin{keyinsight}
Humans do not adapt to delays strategically. The impact is purely additive---no compensation occurs.
\end{keyinsight}

\section{Design Recommendations}

\begin{enumerate}
    \item Use streaming responses
    \item Provide progress indicators
    \item Implement async patterns
    \item Set user expectations appropriately
\end{enumerate}

\section{Key Takeaways}

\begin{enumerate}
    \item Latency degrades interaction quality independently of response quality
    \item Effect is large, consistent, and non-compensable
    \item Design for latency as a first-class constraint
\end{enumerate}
