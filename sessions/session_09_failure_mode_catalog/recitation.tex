\section{Detection Protocols}

\subsection{Brittleness Testing}
\begin{enumerate}
    \item Create 3+ format variations per scenario
    \item Test with identical ground truth
    \item Calculate max-min accuracy spread
    \item Threshold: $>$30pp indicates pattern matching
\end{enumerate}

\subsection{Bias Rate Testing}
\begin{enumerate}
    \item Create balanced test sets (50/50 split)
    \item Calculate FPR and FNR separately
    \item Check for systematic action bias
    \item Threshold: $>$20\% FPR is significant
\end{enumerate}

\section{Testing Suite Structure}

A complete pre-deployment suite includes:
\begin{enumerate}
    \item Brittleness tests (all applications)
    \item Bias tests (action-oriented applications)
    \item Domain-specific failure tests
    \item Latency benchmarks (interactive applications)
    \item Scaling analysis (if model size is a variable)
\end{enumerate}

\begin{keyinsight}
Testing is not optional for production deployment. Each failure mode requires its own detection protocol.
\end{keyinsight}

\section{Key Takeaways}

\begin{enumerate}
    \item Apply the complete taxonomy---don't skip failure modes
    \item Prioritize based on domain-specific risks
    \item Document all testing with pass/fail criteria
    \item Testing results feed into hybrid architecture decisions
\end{enumerate}
